# log-mistake

실수(빌드/테스트 깨짐, 잘못된 가정, 반복 버그, 사용자 피드백 위반)를 영속 메모리에 기록한다.

## 입력

- `symptom` — 무엇이 잘못됐는가(증상/에러)
- `cause` — 근본 원인
- `fix` — 어떻게 고쳤는가
- `prevention` — 재발 방지 규칙(가능하면 CLAUDE.md/스킬에 반영)

## 동작

1. `.claude/memory/MISTAKES.md` 를 읽는다(없으면 헤더와 함께 생성).
2. 다음 포맷으로 **맨 위 최신** 항목 추가:

```markdown
## [YYYY-MM-DD] <한 줄 제목>
- **증상**: ...
- **원인**: ...
- **수정**: ...
- **재발방지**: ...
- **관련 파일**: path:line
```

3. 재발방지 규칙이 일반적이면 루트 `CLAUDE.md` "절대 규칙" 또는 관련 스킬에도 반영 제안.

## 사용 시점

- 테스트/실행 실패를 고친 직후
- 사용자가 "그거 아니야 / 다시 해" 라고 교정했을 때
- 같은 종류의 버그를 두 번째 만났을 때

## 주의

- 시크릿/개인정보(토큰·chat_id)는 기록하지 않는다.
- 코드에서 자명한 사실은 기록하지 않는다 — 반복 위험이 있는 **판단/가정의 오류**만 남긴다.
